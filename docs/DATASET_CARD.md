# Dataset Card

## Dataset Purpose

This dataset supports single-class instance segmentation of a phone cover. The trained segmentation mask is later used for metric width and height measurement.

## Object Selection

| Item | Value |
|------|-------|
| Object | Phone cover |
| Material | Silicone/rubber case |
| Class name | `Phone_Cover` |
| Approximate dimensions | 80 mm width x 170 mm height |

## Object Choice Rationale

- The object is readily available and can be photographed repeatedly.
- It is mostly planar, which is appropriate for a calibrated 2D measurement task.
- Its outline is close to rectangular, making width and height meaningful.
- It normally appears as one connected region, which makes polygon labelling practical.
- The boundary is visually distinct enough for segmentation under varied backgrounds.

## Collection Strategy

| Item | Value |
|------|-------|
| Camera | Same camera/device used for calibration |
| Raw image directory | `dataset/raw/images/` |
| Raw images captured | 92 |
| Undistorted images | 92 |
| Labelled images | 90 |
| Labelled image directory | `dataset/undistorted/images/` |
| Annotation file | `dataset/undistorted/annotations/instances_default.json` |

Images were captured with variation in:

- Distance from camera.
- Top-down and oblique viewing angles.
- Object rotation.
- Background surfaces.
- Indoor natural and artificial lighting.
- Object placement inside the frame.

All dataset images were undistorted before labelling and training to keep the model input distribution aligned with the calibrated inference pipeline.

## Labelling Process

| Item | Value |
|------|-------|
| Tool | CVAT |
| Annotation type | Polygon instance segmentation |
| Export format | COCO 1.0 |
| Instances per labelled image | 1 phone-cover instance |
| Categories | 1 foreground category plus implicit background |

Each labelled image contains one polygon mask around the visible phone cover. COCO annotations include image metadata, segmentation polygons, bounding boxes, area values, and category IDs.

## Dataset Splits

The split was generated by:

```text
dataset/prepare_dataset.py
```

Random seed:

```text
42
```

| Split | Images | Percentage | Annotation file |
|-------|--------|------------|-----------------|
| Train | 62 | 68.9% | `dataset/train/annotations.json` |
| Validation | 19 | 21.1% | `dataset/val/annotations.json` |
| Test | 9 | 10.0% | `dataset/test/annotations.json` |
| Total | 90 | 100.0% | - |

The target split was 70/20/10. Integer rounding on 90 labelled images produces 62/19/9.

## Class Distribution

| Class | Train | Validation | Test | Total |
|-------|-------|------------|------|-------|
| `Phone_Cover` | 62 | 19 | 9 | 90 |

This is a single-class dataset. Class balance is therefore uniform by design.

## Data Quality Notes

- The object is visible as the main target in each labelled image.
- Images include diverse scale, rotation, background, and lighting.
- Images were undistorted before annotation.
- The dataset is small but sufficient for a single-object technical assessment when combined with transfer learning.

## Augmentation Strategy

No explicit online augmentation was applied in the current training loop. The model relies on real capture diversity plus COCO pretrained weights.

Future augmentation options:

- Random horizontal/vertical flips.
- Small rotations.
- Brightness and contrast jitter.
- Random scale or crop.
- Mild blur/noise simulation.

## Dataset Assumptions

- Each image contains one target object instance.
- The target object is not severely occluded.
- Segmentation labels are treated as ground truth for model training and evaluation.
- Undistorted images are the canonical dataset version.

## Dataset Limitations

- Only one foreground class is present.
- The dataset is small for broad real-world generalisation.
- No explicit augmentation is included in training.
- Test metrics are based on only 9 held-out images.
